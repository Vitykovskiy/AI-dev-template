# Glossary

## Business Terms

- Business task intake: structured clarification phase that turns a raw request into an actionable and testable definition of work.
- Epic: a large business or delivery outcome that is decomposed into smaller issues.
- First version: the narrowest delivery scope that still produces meaningful business value.

## Technical Terms

- Canonical source of truth: the authoritative location where project state must be persisted.
- Environment check: validation of local tooling, repository wiring, GitHub access, and baseline readiness.
- Guardrail: a configured category of change that still requires explicit human approval in otherwise autonomous workflows.

## Working Definitions

- Done: implemented, documented, reflected in GitHub state, and committed.
- Ready: sufficiently clarified to begin implementation without guessing core requirements.
- Blocked: cannot continue because of an unresolved external dependency or missing information.
