# Tasks Directory

The canonical backlog lives in GitHub Issues and GitHub Project.

Use this directory only for temporary local notes, rough drafts, or scratch files created while executing one issue-driven task.

Rules:

- Final initiative and task definitions live in GitHub Issues.
- Each operational task must live in its own GitHub Issue.
- Each task must have exactly one owner contour and explicit dependencies.
- Final execution status lives in GitHub Project.
- Final analysis, design, implementation handoff, rollout, and validation decisions live in `docs/`.
- Do not treat files in `tasks/` as the durable backlog or the canonical specification.
